# 数据验证方法论

## 五级顺序验证法（规则一致性分级）

| 级别 | 数据来源 | 适用范围 | 验证重点 |
|------|---------|---------|---------|
| 第1级 | 公司官方财报（10-K/年报/交易所公告） | 全部字段 | 若已查询到有效数据，应直接采用 |
| 第2级 | 权威财经数据库/接口 | 全部字段 | 仅当第1级无有效数据时进入 |
| 第3级 | 行业机构/研究报告 | 全部字段 | 仅当前两级无有效数据时进入 |
| 第4级 | 财经媒体/全网搜索 | 全部字段 | 仅当前三级无有效数据时进入 |
| 第5级 | 估算/预测/人工分析 | 全部字段 | 仅当前四级无有效数据时允许使用 |

核心验证目标不是偏好某一类来源，而是验证：**是否按五级顺序查询、是否在首次命中时即使用、是否正确标注来源级别或数据缺口。**

## 数据验证五步法

```
Step 1: 识别数据项是否已有有效值
- 若字段已按更高优先级查询到有效数据 → 直接采用，不再继续下探
- 若字段当前无有效数据 → 继续进入下一级来源

Step 2: 核查是否遵守五级顺序
- 必须按 第1级 > 第2级 > 第3级 > 第4级 > 第5级 逐级查询
- 不得根据字段特性跳级，也不得在命中后再改选来源

Step 3: 核查原始来源
- 第1级：SEC EDGAR / HKEXnews / 巨潮 / 公司IR
- 第2级：AKShare / 东方财富 / 同花顺 / 新浪等
- 第3级：行业机构 / 券商研报
- 第4级：财经媒体 / ProSearch结果
- 第5级：估算 / 预测 / 人工分析

Step 4: 明确标注来源（禁止使用emoji标志）
- 标注为"来源级别:第1级" 到 "来源级别:第5级"
- 无法获取时标注"数据缺口"
- 必须同时说明具体来源、日期或口径

Step 5: 输出修正对比表（如有错误）
- 初始值 vs 修正值 vs 实际值
- 明确差异、修正原因和最终采用的来源级别
```

## 常见数据错误类型

| 错误类型 | 典型案例 | 预防措施 |
|---------|---------|---------|
| 用估算代替官方数据 | 用"约4500亿"代替官方财报精确值 | 必须查阅官方财报 |
| 时间混淆 | 将季度累计当全年 | 确认财报时间范围 |
| 单位混淆 | 亿/亿美元/百万美元 | 明确标注货币单位 |
| 概念混淆 | 营收/净利润/毛利 | 核对定义 |
| 估算当真 | 市场份额当官方数据 | 明确标注数据来源 |
| 来源不可靠 | 用媒体二手数据 | 优先查阅原始财报 |
| 不标注来源 | 只写数字不写来源 | 每一项数据必须标注来源 |

## 处理用户质疑的正确姿势

```
1. 主动承认错误（不要辩解）
2. 立即核查数据（官方财报数据库 + 精确搜索关键词 + 多源交叉验证）
3. 明确区分"准确数据"和"估算数据"（用文字说明，禁止emoji）
4. 输出修正对比表：| 数据项 | 错误值 | 正确值 | 差异 | 来源 |
5. 重新生成准确文件（使用Python脚本，标注每一项数据来源）
```

## 实战案例：NVIDIA FY2025

| 数据项 | 错误值 | 正确值 | 来源 |
|--------|--------|--------|------|
| 总资产 | 估算600-700亿 | $1,116亿 | 雪球财报解析 |
| ROE | 估算40-60% | 119.18% | 雪球财报解析 |
| 净利润 | 未提供 | $728.8亿 | 雪球财报解析 |

**关键教训**：用户质疑往往是对的；诚实报告错误比掩盖更重要；核心财务数据必须用官方财报。
