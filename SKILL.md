## 标准工作流程

### Step 1: 确认公司信息 + 年报发布状态检查

确认:1) 公司名称(中英文)2) 股票代码 3) 财务年度(默认最新年报)

**【必须首先执行】年报发布状态检查**:

> **问题根源**:不先确认年报是否发布,直接使用"年报显示XXX"的表述,导致混淆业绩预告数据、年报数据和季报数据。

在提取任何数据之前,**必须先通过搜索确认年报发布状态**:

```bash
node prosearch.cjs --keyword="{公司名} {股票代码} {年份}年报 正式发布"
```

**发布状态对应数据策略**:

| 发布状态 | 数据来源优先级 | 必须标注内容 |
|---------|--------------|------------|
| **已发布** | 官方年报全文(公司官网/巨潮资讯/东方财富) | "年报显示(发布日期)" |
| **未发布** | 业绩预告/业绩快报/季报 | "数据来源:{具体来源},年报尚未发布,最终数据可能与年报有差异" |
| **仅业绩预告** | 业绩预告数据(通常含营收±5%、净利润±20%范围) | "预测数据(业绩预告区间中位数)" |
| **仅季报** | 最近一期季报(如Q3已出但年报未出) | "数据截至{Q3}季报(YYYY-MM-DD),年报尚未发布" |

**常见错误 vs 正确做法**:

| 场景 | 错误表述 | 正确表述 |
|------|---------|---------|
| 年报已发布 | "年报显示营收1178亿" | "年报显示营收1178亿(2026-04-13发布)" |
| 仅业绩预告 | "年报显示营收1178亿" | "业绩快报显示营收1178亿,年报尚未发布" |
| 仅Q3季报 | "年报显示前三季度营收874亿" | "Q3季报显示前三季度营收874亿(截至Q3),年报尚未发布" |

**验证通过标准**:
- 搜索结果出现"发布{年份}年度报告"或"年报披露"字样
- 找到具体发布日期(如"2026-04-13")
- 才能使用"年报显示"等表述

**搜索命令优先级**:

**【容错】AKShare 失败时自动切换到 ProSearch**

> **问题**:AKShare 连接东方财富接口经常失败(网络代理、超时、限流),导致数据提取中断。

**自动容错流程**:
```python
from akshare_universal_integration import AKShareUniversalReport

extractor = AKShareUniversalReport()

# Step 1: 尝试 AKShare
try:
    data = extractor.extract_report_data("A股", "600710")
    if data and len(data) > 0:
        print("AKShare 数据获取成功")
        # 使用 AKShare 数据
    else:
        raise ValueError("AKShare 返回空数据")
except Exception as e:
    print(f"AKShare 失败: {e}")
    # 自动切换到 ProSearch
    print("自动切换到 ProSearch 全网搜索...")
```

**AKShare 失败场景识别**:

| 错误类型 | 原因 | 切换策略 |
|---------|------|---------|
| `ProxyError` / `ConnectTimeout` | 网络代理问题 | 立即切换 ProSearch |
| `HTTPSConnectionPool Max retries` | 接口超时 | 立即切换 ProSearch |
| `NoneType object is not subscriptable` | 数据解析失败 | 立即切换 ProSearch |
| 返回空数据 | 接口无返回 | 立即切换 ProSearch |
| 限流 429 | 访问过于频繁 | 等待10秒后重试1次,仍失败则切换 ProSearch |

**ProSearch 备选搜索命令**:
```bash

node prosearch.cjs --keyword="{公司名} {股票代码} {年份}年报 总资产 净资产 负债率"

```

```python
def get_financial_data(stock_code, year):
    """带容错的数据获取函数"""
```

### Step 3: 数据提取顺序

**强制规则:所有字段都必须按统一五级信息优先级逐级查询,且哪一级先查询到有效数据,就直接使用哪一级的数据。**

无论字段属于核心财务、行情估值、行业对比、供应链、股权变动还是未来预测,都不得只固定使用某一种来源,也不得在已经查询到有效数据后,再根据字段特性改选所谓"更合适"的来源。每一个字段都必须按以下顺序依次查询、核验和标注:

| 级别 | 信息来源 | 可靠性 | 适用范围 | 处理规则 |
|------|----------|--------|----------|----------|
| 第1级 | 官方/监管披露数据:公司官网IR、年报、半年报、季报、业绩公告、HKEXnews、巨潮资讯、SEC EDGAR、交易所公告 | 最高 | 全部字段 | 第1级一旦查询到该字段的有效数据,立即使用并停止继续向下查询 |
| 第2级 | 权威财经数据库/接口:AKShare、东方财富、同花顺、新浪财经、Wind、Bloomberg、FactSet、Refinitiv、交易行情接口 | 高 | 全部字段 | 仅当第1级未查询到有效数据时才进入第2级; 一旦查询到即使用 |
| 第3级 | 权威行业机构/研究报告:IDC、Omdia、Canalys、Counterpoint、Gartner、艾瑞咨询、智研咨询、中商情报网、券商研报、行业白皮书 | 中高 | 全部字段 | 仅当前两级未查询到有效数据时才进入第3级; 一旦查询到即使用 |
| 第4级 | 权威财经媒体/年报解读/全网搜索:证券之星、雪球、中证智能财讯、财联社、21财经、搜狐财经、腾讯财经、新浪财经新闻、ProSearch结果 | 中 | 全部字段 | 仅当前三级未查询到有效数据时才进入第4级; 一旦查询到即使用 |
| 第5级 | 估算/预测/人工分析:分析师预测、行业估算、模型测算、可比公司推导、人工推断 | 低 | 全部字段 | 仅当前四级均未查询到有效数据时才允许使用; 若仍无结果则标注数据缺口 |

**字段级执行要求:**
1. 对22项数据表中的每一个字段,都必须严格按第1级 → 第2级 → 第3级 → 第4级 → 第5级的顺序查询。
2. 哪一级先查询到该字段的有效数据,就直接使用哪一级数据,不得继续向下查询后再按字段特性改选来源。
3. 只有在当前级别未查询到有效数据时,才能进入下一级。
4. 所谓"有效数据"是指来源可追溯、字段口径明确、可写入Excel且能说明来源级别的数据; 若仅有模糊线索,不算有效数据,应继续进入下一级。
5. 如果同一级别查到多个结果冲突,优先使用发布时间更新、口径更完整、可追溯性更强的来源。
6. 实时数据必须标注数据日期; 预测数据必须说明预测口径; 无法获取的数据必须标注"数据缺口"。
7. 表1 D列必须写清该字段最终采用了哪一级来源、具体来源名称、可靠性等级和缺口原因。

### Step 4: 数据验证(核心流程)

**核心原则:核心财务数据必须用官方财报,不能用估算!**

#### 4.1 全字段五级数据来源优先级(强制遵守)

**本规则适用于全部22项字段和数据表2中的所有分析依据。** 不允许只对核心财务字段使用高优先级来源,也不允许对行业、供应链、估值、预测字段直接跳到低优先级估算。**所有字段严格执行"按级别顺序查询,哪一级先查询到有效数据,就使用哪一级数据"的规则,不得再根据字段特性选择所谓更合适的最终来源。**

| 优先级 | 数据来源 | 可靠性 | 用途 | 强制处理规则 |
|--------|---------|--------|------|--------------|
| 1 | 官方/监管披露数据:公司官网IR、年报、半年报、季报、业绩公告、交易所公告、SEC EDGAR、HKEXnews、巨潮资讯网 | 最高 | 全部字段的首选来源 | 第1级查询到有效数据即直接采用,停止继续向下查询 |
| 2 | 权威财经数据库/接口:AKShare、东方财富、同花顺、新浪财经、Wind、Bloomberg、FactSet、Refinitiv、交易行情接口 | 高 | 全部字段 | 仅当第1级未查询到有效数据时进入; 一旦查询到即直接采用 |
| 3 | 权威行业机构/研究报告:IDC、Omdia、Canalys、Counterpoint、Gartner、艾瑞咨询、智研咨询、中商情报网、券商研报、行业白皮书 | 中高 | 全部字段 | 仅当前两级未查询到有效数据时进入; 一旦查询到即直接采用 |
| 4 | 权威财经媒体/年报解读/全网搜索:证券之星、雪球、中证智能财讯、财联社、21财经、搜狐财经、腾讯财经、新浪财经新闻、ProSearch结果 | 中 | 全部字段 | 仅当前三级未查询到有效数据时进入; 一旦查询到即直接采用 |
| 5 | 估算/预测/人工分析:分析师预测、行业估算、模型测算、可比公司推导、人工推断 | 低 | 全部字段 | 仅前四级均未查询到有效数据时才允许使用; 若第5级也无法形成有效数据则标注数据缺口 |

**禁止行为**:
- 某字段未按第1-5级顺序逐级查询
- 在较高优先级已经查询到有效数据后,继续向下查询并改选更低优先级来源
- 根据字段特性二次挑选所谓"更合适"来源,绕开"先查询到先使用"规则
- 用媒体二手数据代替官方财报核心数字
- 用估算当官方数据
- 不标注来源和来源级别
- 混淆官方数据、权威来源、实时数据、行业估算、预测数据和人工推测
- 不说明数据缺口原因

## 数据来源标注规范(禁止emoji)

每个字段的D列必须同时说明"五级来源级别 + 具体来源 + 可靠性 + 口径/日期/缺口原因"。

- **第1级 官方/监管披露数据**:来自公司IR、年报、10-K、季报、业绩公告、交易所公告、HKEXnews、巨潮资讯、SEC EDGAR。标注为"来源级别:第1级 | 可靠性:官方/监管披露"。
- **第2级 权威财经数据库/接口**:来自AKShare、东方财富、同花顺、新浪财经、Wind、Bloomberg、FactSet、Refinitiv或行情接口。实时行情必须标注"数据日期:YYYY-MM-DD"。
- **第3级 权威行业机构/研究报告**:来自IDC、Omdia、Canalys、Counterpoint、Gartner、券商研报、行业白皮书等。标注为"可靠性:权威来源"或"行业估算",并说明报告/机构口径。
- **第4级 权威财经媒体/全网搜索补充**:来自证券之星、雪球、中证智能财讯、财联社、21财经、搜狐财经、腾讯财经、新浪财经新闻或ProSearch结果。必须说明其仅为补充或线索,核心数字需回溯核验。
- **第5级 估算/预测/人工分析**:来自分析师预测、模型测算、可比公司推导或人工推断。必须说明"预测数据/行业估算/分析推测"; 如果仍无法获取,标注"数据缺口"并说明原因和建议补充来源。

```
可靠性:官方/监管披露数据

来源级别:第2级
来源:东方财富港股实时行情

数据日期:2026-04-25

来源级别:第3级
来源:Omdia 2025智能手机出货量统计(公司公告引用)
可靠性:权威行业机构; 若由公司公告引用,同时说明第1级官方披露引用第三方数据

来源级别:第5级
来源:可比公司样本测算
可靠性:行业估算
说明:官方未披露行业平均ROE,按可比公司公开财报口径测算
```

## 注意事项

1. **市场识别**:根据股票代码自动识别市场(A股6位数字、港股5位数字、美股字母代码)
2. **货币单位统一**:A股用RMB、港股用HKD、美股用USD,必须标注
3. **时间口径**:确认财年周期(自然年 vs FY),美股财年可能不按自然年,必须标注
4. **行业对比**:确保口径一致。所有字段都严格按五级顺序查询,哪一级先查询到有效数据就使用哪一级; 如果最终使用行业均值或行业机构数据,需标注"行业估算"或相应来源级别
5. **供应链数据**:同样适用五级优先级; 必须按第1级到第5级顺序逐级查询,哪一级先查询到有效数据就使用哪一级,仍无法获取则标注"数据缺口"
6. **用户质疑处理**:主动承认错误 → 核查数据 → 区分官方/估算 → 输出修正对比表 → 重新生成
7. **全网搜索修复**:初始提取后仍有行业估算/数据缺失项 → 执行Step 5全网搜索修复流程 → 6轮ProSearch将估算项从8-10个降至2-3个 → 仍无法获取的标注"数据缺口"
8. **港股/美股数据**:AKShare提供核心财务数据,部分指标可能需与官方10-K/年报交叉验证
9. **年报发布状态检查(最重要!)**:在提取任何公司年报数据前,必须首先搜索确认年报是否已发布。已发布才用"年报显示",未发布必须标注数据来源(业绩预告/季报)和时效。
10. **行业数据**:同样适用五级优先级; 必须按第1级到第5级顺序逐级查询,哪一级先查询到有效数据就使用哪一级。若最终使用行业报告、样本测算或第三方机构数据,需标注来源级别与测算口径
